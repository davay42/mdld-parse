# PROV-O Provenance Demo {=my:prov-demo .prov:Entity label}

Demonstrates complete provenance patterns using W3C PROV-O ontology.

## Core PROV-O Classes

### Entity Example {=my:report .prov:Entity label}

A report is an entity that exists and can be described.

Generated by: [Analysis Activity] {+my:analysis ?prov:wasGeneratedBy}
Attributed to: [Alice] {+my:alice .prov:Person ?prov:wasAttributedTo}
Derived from: [Raw Data] {+my:raw-data ?prov:wasDerivedFrom}

### Activity Example {=my:analysis .prov:Activity label}

An activity is something that happens over time.

Started: [2026-05-06T09:00:00Z] {prov:startedAtTime ^^xsd:dateTime}
Ended: [2026-05-06T10:30:00Z] {prov:endedAtTime ^^xsd:dateTime}
Associated with: [Alice] {+my:alice ?prov:wasAssociatedWith}
Used: [Raw Data] {+my:raw-data ?prov:used}
Generated: [Report] {+my:report ?prov:generated}

### Agent Example {=my:alice .prov:Person label}

An agent is something that can bear responsibility.

Name: [Alice Smith] {my:name}
Email: [alice@example.com] {my:email}
Role: [Data Analyst] {my:role}

## Qualified Provenance Patterns

### Qualified Usage {=my:qualified-usage .prov:Usage ?prov:qualifiedUsage label}

Describes how an entity was used in an activity.

Entity: [Raw Data] {+my:raw-data ?prov:entity}
Activity: [Analysis] {+my:analysis ?prov:activity}
Role: [Input Dataset] {+my:input-dataset .prov:Role ?prov:hadRole}
Time: [2026-05-06T09:15:00Z] {prov:atTime ^^xsd:dateTime}

### Qualified Generation {=my:qualified-generation .prov:Generation ?prov:qualifiedGeneration label}

Describes how an entity was generated by an activity.

Entity: [Report] {+my:report ?prov:entity}
Activity: [Analysis] {+my:analysis ?prov:activity}
Role: [Final Output] {+my:output .prov:Role ?prov:hadRole}
Time: [2026-05-06T10:30:00Z] {prov:atTime ^^xsd:dateTime}

### Qualified Association {=my:qualified-association .prov:Association ?prov:qualifiedAssociation label}

Describes agent's role in an activity.

Activity: [Analysis] {+my:analysis ?prov:activity}
Agent: [Alice] {+my:alice ?prov:agent}
Role: [Lead Analyst] {+my:lead-analyst .prov:Role ?prov:hadRole}
Plan: [Q2 Analysis Plan] {+my:q2-plan .prov:Plan ?prov:hadPlan}

## Delegation Pattern

### Delegation Example {=my:delegation .prov:Delegation ?prov:qualifiedDelegation label}

Alice delegates authority to Bob for a specific task.

Delegate: [Alice] {+my:alice ?prov:delegate}
Responsible: [Bob] {+my:bob .prov:Person ?prov:responsible}
Activity: [Data Review] {+my:review .prov:Activity ?prov:hadActivity}
Role: [Reviewer] {+my:reviewer .prov:Role ?prov:hadRole}
